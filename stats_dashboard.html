Pipeline Calculations
Current Settings
  • Min turns: {{ pipeline_settings.min_turns }} — dialogues with fewer turns are excluded as test/unmeaningful
  • Min reviews required: {{ pipeline_settings.min_reviews_required }} — reviews needed before the rating threshold is checked
  • Rating threshold: {{ pipeline_settings.rating_threshold }} — average rating needed to count as "Rating Reached"
Top-level Cards
  • Total Reviews: Count of all human peer review submissions (LLM reviews excluded). Counts across all versions (V1, V2, V3) and all dialogues regardless of turn count.
  • V1 Dialogues Created: V1 Completed + In Progress + Test/Unmeaningful. This is every V1 dialogue in the system.
  • Rating Change (V1 to V2): Avg V2 Rating minus Avg V1 Rating. Each dialogue's average is computed first, then those averages are averaged (per-dialogue, not per-review).
Flowchart Stages Cumulative

Each stage counts all dialogues that passed through it, including those that progressed further. Percentages show stage-to-stage conversion rates.

  • Test/Unmeaningful: Dialogues (any version) with fewer than {{ pipeline_settings.min_turns }} turns or invalid content. Excluded from the pipeline entirely.
  • In Progress: V1 dialogues where in_progress = True. Not yet submitted.
  • Dialogue V1 Completed: Meaningful V1 dialogues that are not in progress. This is the pipeline entry point and the denominator for Cumulative Conversion.
  • Review 1 (V1) Completed: V1 dialogues with at least 1 review.
  • Review 2 (V1) Completed: V1 dialogues with at least {{ pipeline_settings.min_reviews_required }} reviews. The average rating is calculated at this point.
  • Rating Reached (gate): After Review 2, if a dialogue's average rating is ≥ {{ pipeline_settings.rating_threshold }}, it exits the pipeline as "Rating Reached". The family is not tracked further.
  • Edit (V1) Completed: V1 dialogues that did not reach {{ pipeline_settings.rating_threshold }} and have a V2 version created.

The same pattern repeats for V2 and V3. After V3 Review 2, dialogues that still don't reach {{ pipeline_settings.rating_threshold }} are counted as "Rating Not Reached".

Table
  • Count: Total dialogues that reached this stage (includes those that moved on).
  • Cumulative Conversion: Stage count / Dialogue V1 Completed × 100.
  • Stage Conversion: Stage count / previous stage count × 100. Shows the drop-off between consecutive stages.
Flowchart Stages Unique

Each stage counts only dialogues whose furthest progress is that stage — they have not moved beyond it. Useful for identifying where dialogues are stuck.

  • Dialogue V1 Completed: Submitted V1 dialogues that have not received any reviews yet.
  • Review 1 (V1) Completed: V1 dialogues with 1 review but fewer than {{ pipeline_settings.min_reviews_required }}. Waiting for more reviews.
  • Review 2 (V1) Completed: V1 dialogues with {{ pipeline_settings.min_reviews_required }}+ reviews whose rating is below {{ pipeline_settings.rating_threshold }}, but no V2 has been created yet. Waiting for the author to edit.
  • Edit (V1) Completed: A V2 exists but has not received any reviews yet.

The same pattern repeats for V2 and V3. Terminal stages (Rating Reached, Not Reached) are the same in both views.

How unique stage is determined

Each dialogue family (V1 + V2 + V3 sharing the same root) is traced through the pipeline. The furthest stage it reaches is the one it counts toward. For example, a family with V1 that has 1 review and no V2 counts uniquely in "Review 1 (V1) Completed". The sum of all unique counts equals the total V1 Completed count.

Table
  • Count: Dialogues whose furthest progress is exactly this stage.
  • % of Total: Unique count / Dialogue V1 Completed × 100. Shows what fraction of all dialogues are at this stage.
Average Ratings (Sidebar)
  • Avg V1 Rating: Only V1 dialogues with 2 completed reviews are included. Each dialogue's average rating is computed, then all per-dialogue averages are averaged together. Dialogues with 0 or 1 reviews are excluded.
  • Avg V2 Rating: Same calculation for V2 dialogues.
  • Avg V3 Rating: Same calculation for V3 dialogues.
Family Grouping

Dialogues are grouped into families by parent_dialogue_id. All versions (V1, V2, V3) of the same dialogue share the same root ID. The pipeline tracks each family through its lifecycle as a single unit. A family exits the pipeline the first time it reaches the rating threshold.
